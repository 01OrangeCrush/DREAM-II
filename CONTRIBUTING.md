# Contributing

Contributions should improve reproducibility, documentation, analysis quality, or accessibility without exposing human-participant information or redistributing restricted source materials.

Before opening a pull request:

1. Remove names, emails, credentials, and reviewer-code linkage information.
2. Confirm that no student records or private educational data are included.
3. Preserve the frozen analytic prompt; proposed prompt changes must be placed in a new versioned file.
4. Document changes in `CHANGELOG.md`.
5. Add or update tests/checks for analysis code where applicable.
6. Cite original artifact sources and licenses.

Report possible privacy or research-integrity concerns privately to the project contact rather than in a public issue.
